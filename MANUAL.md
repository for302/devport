## 1. 개요

### 1.1 DevPort Manager란?

DevPort Manager V2는 Windows 환경에서 웹 개발에 필요한 모든 도구와 서비스를 통합 관리하는 데스크톱 애플리케이션입니다. XAMPP, Laragon 등 기존 개발 환경 도구들의 기능을 현대적인 UI/UX와 함께 제공합니다.

### 1.2 주요 기능

- **서비스 관리**: Apache, MariaDB 등 개발 서버 실시간 제어
- **프로젝트 관리**: 다양한 프레임워크 프로젝트 등록 및 실행 관리
- **컴포넌트 설치**: Node.js, PHP, Python 등 런타임 및 도구 자동 설치
- **데이터베이스 관리**: MariaDB 데이터베이스 생성 및 phpMyAdmin 연동
- **환경 설정**: .env 파일 GUI 편집기, hosts 파일 자동 관리
- **포트 관리**: 시스템 포트 스캔 및 충돌 감지
- **로그 시스템**: 실시간 로그 스트리밍 및 로테이션

### 1.3 기술 스택

| 구분 | 기술 |
|------|------|
| 데스크톱 프레임워크 | Tauri 2.x (Rust + WebView) |
| 프론트엔드 | React 19 + TypeScript + Vite |
| UI 스타일링 | Tailwind CSS 4 |
| 상태 관리 | Zustand with Immer |
| 아이콘 | Lucide React |

## 3. 설치 및 초기 설정

### 3.1 설치 과정

1. **설치 파일 실행**: `ClickDevPort_x.x.x_x64-setup.exe` 실행
2. **설치 위치 확인**: 기본 설치 경로 `C:\DevPort\` 확인
3. **설치 완료**: 바탕화면 및 시작 메뉴에 바로가기 생성

### 3.2 초기 설정 마법사

첫 실행 시 설치 마법사가 나타납니다:

#### Step 1: 프리셋 선택
개발 환경에 맞는 프리셋을 선택합니다:
- **웹 개발 기본**: Apache + MariaDB + PHP + Node.js
- **프론트엔드 전용**: Node.js + 빌드 도구
- **풀스택**: 모든 주요 컴포넌트
- **사용자 정의**: 개별 컴포넌트 선택

#### Step 2: 컴포넌트 선택
프리셋에 포함된 컴포넌트를 확인하고 추가/제거합니다.

#### Step 3: 설치 진행
- 다운로드 진행률 표시
- 컴포넌트별 설치 상태
- 실시간 로그 출력

#### Step 4: 완료
설치된 컴포넌트 목록 확인 후 앱 사용 시작

### 3.3 건너뛰기

설치 마법사는 "건너뛰기 (나중에 설치)" 버튼으로 나중에 진행할 수 있습니다.

## 6. 서비스 관리 (Services)

### 6.1 개발 환경 인벤토리

시스템에 설치된 모든 개발 도구를 스캔하여 표시합니다.

#### 카테고리

| 카테고리 | 예시 |
|----------|------|
| Web Servers | Apache, Nginx |
| Databases | MySQL/MariaDB, PostgreSQL, Redis, MongoDB |
| Runtimes | Node.js, Python, PHP, Java, Go, Ruby |
| Build Tools | npm, pip, Composer, Maven |
| Frameworks | React, Vue, Django, Laravel |
| Package Managers | npm, yarn, pnpm, pip |
| Dev Tools | Git, Docker, VS Code |

#### 감지 소스

인벤토리 스캔은 다음 설치 소스를 감지합니다:
- System (시스템 PATH)
- DevPort (본 앱으로 설치)
- XAMPP
- Laragon
- WampServer
- Scoop
- Chocolatey
- 수동 설치

### 6.2 인벤토리 표시 정보

| 정보 | 설명 |
|------|------|
| 이름 | 컴포넌트 이름 |
| 버전 | 설치된 버전 |
| 설치 소스 | 어디서 설치되었는지 |
| 실행 경로 | 실행 파일 위치 |
| 크기 | 설치 용량 |

### 6.3 검색 및 필터

- **검색**: 이름으로 빠른 검색
- **필터**: "설치됨만 표시" 토글
- **재스캔**: 인벤토리 다시 스캔

### 6.4 서비스 상태 모델 (4단계)

| 상태 | 아이콘 | 의미 |
|------|--------|------|
| Running | 🟢 | PID 존재 + Health Check 통과 |
| Stopped | ⚫ | PID 없음 |
| Error | 🔴 | 시작 실패 또는 크래시 |
| Unhealthy | 🟡 | PID 존재 + Health Check 실패 |

### 6.5 서비스 제어

**Apache HTTP Server**
- 기본 포트: 8080
- 제어: 시작 / 중지 / 재시작
- 상태: PID, 포트, Health Check

**MariaDB**
- 기본 포트: 3306
- 제어: 시작 / 중지 / 재시작
- 상태: PID, 포트, Health Check

### 6.6 phpMyAdmin 상태

- 상태 배지: Available / Unavailable / Checking
- 응답 시간 측정
- Apache 실행 시 브라우저에서 열기 버튼

## 7. 포트 관리 (Ports)

### 7.1 포트 스캐너

시스템의 모든 활성 네트워크 포트를 스캔하여 표시합니다.

### 7.2 표시 정보

| 컬럼 | 설명 |
|------|------|
| Port | 포트 번호 |
| PID | 프로세스 ID |
| Process | 프로세스 이름 |
| Protocol | TCP / UDP |
| State | LISTENING, ESTABLISHED 등 |

### 7.3 기능

- **수동 새로고침**: 버튼 클릭으로 재스캔
- **마지막 스캔 시간**: 최근 스캔 시점 표시
- **포트 충돌 감지**: 프로젝트 생성 시 자동 확인

### 7.4 포트 가용성 확인

프로젝트 추가/편집 시:
- 🟢 Available: 포트 사용 가능
- 🟡 In use: 다른 프로세스가 사용 중

| Git | C:\DevPort\git |

⚠️ 시스템에 이미 설치된 버전과 충돌할 수 있습니다.

### 9.4 로그 설정

  - 최소 8자 이상

- **기본 백업 위치**: 백업 저장 경로 설정

- 설치된 도구 배지 (색상으로 구분)
- 상세 목록 (이름, 버전, 소스)
- "설치 시작" 버튼

### 10.2 설치 마법사

| 시작 명령어 | scripts.dev 또는 프레임워크 기본값 |
| 기본 포트 | 설정 파일 또는 프레임워크 기본값 |

| 가상환경 | venv, .venv, env 폴더 감지 |

- 시작 명령어

### 11.5 프로젝트 삭제

3. 자동으로:
   - 프로젝트 목록에서 제거
   - hosts 파일 항목 정리 (도메인 설정된 경우)

⚠️ 실제 프로젝트 파일은 삭제되지 않습니다.

### 11.6 커스텀 도메인

프로젝트에 커스텀 도메인을 설정하면:
- hosts 파일에 자동 추가 (127.0.0.1 매핑)
- 지원 TLD: `.test`, `.localhost`, `.local`

예시:
- `my-project.test` → `127.0.0.1`
- `api.localhost` → `127.0.0.1`

### 11.7 GitHub 연동

프로젝트의 GitHub 저장소 연결:
- `.git/config`에서 자동 감지
- SSH/HTTPS URL 모두 지원
- GitHub 아이콘으로 빠른 접근
- 미연결 시 아이콘 비활성화

예시:
- 프로젝트: `my-blog`

## 18. 문제 해결

### 18.1 서비스가 시작되지 않음

**Apache**:
1. 포트 충돌 확인 (Ports 탭)
2. 다른 웹 서버 확인 (IIS, nginx 등)
3. 로그 확인 (Logs 탭)

**MariaDB**:
1. 포트 3306 사용 확인
2. 다른 MySQL 인스턴스 확인
3. 데이터 디렉토리 권한 확인

### 18.2 프로젝트가 실행되지 않음

1. 시작 명령어 확인
2. Node.js/Python 등 런타임 설치 확인
3. 프로젝트 경로 확인
4. 의존성 설치 확인 (npm install 등)

### 18.3 포트 충돌

1. Ports 탭에서 충돌 포트 확인
2. 해당 프로세스 종료 또는
3. 프로젝트 포트 변경

### 18.4 Hosts 파일 수정 실패

1. 관리자 권한으로 앱 실행
2. 안티바이러스 예외 추가
3. hosts 파일 읽기 전용 해제

### 18.5 설치 실패

1. 인터넷 연결 확인
2. 방화벽 설정 확인
3. 저장공간 확인
4. 설치 로그 확인

### 18.6 초기화

설정 초기화:
1. Settings → "Reset to Defaults" 클릭
2. 앱 재시작

완전 초기화:
1. `%APPDATA%\devport-manager\` 폴더 삭제
2. 앱 재시작
